PR #68 Bugfix

Antes

OwnerSection, ProgramSection e OrigemTypeSection permitiam exclusão sem verificar relacionamentos. Database tinha restrições de cascade/restrict que causavam erros ou exclusões indesejadas.

Depois

OwnerSection mostra aviso sobre contas vinculadas que serão excluídas em cascata. ProgramSection e OrigemTypeSection bloqueiam exclusão quando há dados vinculados, com mensagens explicativas.

Benefícios

  • ✅ Prevenção de exclusão acidental de dados vinculados
  • ✅ Alinhamento com restrições do database
  • ✅ Mensagens claras quando exclusão é bloqueada
  • ✅ Experiência mais segura para o usuário

Database Schema

accounts.owner_id → owners.id (on delete cascade)
accounts.program_id → programs.id (on delete restrict)
entries.origem_type_id → origem_types.id (on delete restrict)
