🔧 PR #86 — Editar Vendas

2026-07-11

feature PR #86 production +125 / -14 linhas

Antes

  • Vendas só podiam ser criadas, canceladas ou ter status alterado
  • Não havia como corrigir dados de uma venda (passageiros, localizador, milhas, valor)

  • Para alterar algo, o usuário precisava cancelar e criar do zero

Depois

  • Botão "Editar" em cada venda não-cancelada na tabela (desktop + mobile)
  • SaleForm suporta mode="edit" com preenchimento automático dos dados
  • Recálculo de lucro e margem automático ao salvar
  • Formulário reinicia corretamente entre edições (key no componente)

Arquivos modificados

  • src/components/SaleForm.tsx — mode "edit" + initialData
  • src/components/SaleTable.tsx — onEdit + botão Editar
  • src/pages/Vendas.tsx — estado editingSale + handleUpdateSale

Benefícios

  • Elimina retrabalho: usuário não precisa cancelar + recriar para corrigir dados
  • UX consistente com o padrão de edição de Entradas
  • Código reutiliza SaleForm existente (sem duplicação)
