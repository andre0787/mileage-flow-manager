🐛 Bugfix: Limpar Cache disparava ErrorBoundary

2026-07-10 | bug fix

Sintoma

Ao clicar em "Limpar Cache" na aba Preferências de /configuracoes, o app exibia:

"Algo deu errado
Ocorreu um erro inesperado ao carregar esta seção. Tente recarregar a página ou clique no botão abaixo."

O ErrorBoundary aparecia por ~100ms antes do window.location.reload() completar.

🔍 Root Cause

Sequência do bug

1. Usuário clica "Limpar Cache"
2. queryClient.clear() — invalida TODAS as queries
3. React re-renderiza com cache vazio
4. Todas as queries disparam refetch simultâneo
5. Se alguma query falha (rede, token, etc), ErrorBoundary captura
6. window.location.reload() — recarrega a página

Problema: Entre os passos 2 e 6, há uma janela de ~100ms onde o React renderiza estado de erro.

🔧 Fix

Antes (DataContext.tsx)

const clearCache = () => {
const keys = Object.keys(localStorage)
.filter(k => k.startsWith("mc-") || k === "mc-migrated");
keys.forEach(k => localStorage.removeItem(k));
queryClient.clear();
window.location.reload();
};

Depois

const clearCache = () => {
const keys = Object.keys(localStorage)
.filter(k => k.startsWith("mc-") || k === "mc-migrated");
keys.forEach(k => localStorage.removeItem(k));
// ponytail: reload descarta cache in-memory, queryClient.clear() só causava re-render com erro
window.location.reload();
};

Raciocínio: window.location.reload() já descarta todo o cache in-memory. queryClient.clear() antes do reload só introduz um estado inconsistente. A localStorage é limpa antes do reload, que é o que persiste.
