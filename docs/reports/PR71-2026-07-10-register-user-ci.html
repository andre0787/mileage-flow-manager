Antes

6 arquivos de teste dependiam de process.env.TEST_EMAIL / TEST_PASSWORD para login. Sem essas variáveis configuradas, 47 testes E2E (auth.spec.ts, clientes.spec.ts, vendas.spec.ts, transversal.spec.ts, configuracoes.spec.ts, smoke.spec.ts) pulavam silenciosamente.

101 linhas de código repetido — cada arquivo tinha o mesmo bloco de login: if (!email || !password) test.skip(); await page.goto('/login') ...

Depois

registerUser(page) em tests/helpers.ts — registro inline via fluxo "Cadastre-se", email dinâmico e2e_{timestamp}_{random}@teste.com. Mesmo padrão dos 8 testes antigos (entradas, clube, carrinho, debug, fluxo-completo, origem-tipo, relatorio, responsivo).

6 arquivos refatorados para usar o helper — 101 linhas a menos. Zero dependência de env vars. Zero testes pulando. Todos os 54 E2E rodam em qualquer ambiente com `npm run test:e2e`.

Benefícios

  • ✅ 47 testes que pulavam agora rodam sempre — cobertura E2E real
  • ✅ Zero dependência externa (sem CI secrets, sem env vars)
  • ✅ -101 linhas — DRY com helper compartilhado
  • ✅ Mesmo padrão dos testes antigos — consistência
  • ✅ CI pode rodar sem configurar secrets
