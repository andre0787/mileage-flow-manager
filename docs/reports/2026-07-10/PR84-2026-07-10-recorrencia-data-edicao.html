📁 +12 arquivos · ➕ 702 linhas · ➖ 22 linhas

⚡ ~965 tokens

Antes

Não havia campo de data na criação/edição de entradas (usava data atual); não havia controle de recorrência; valores não podiam ser distribuídos em parcelas; somente lançamento único.

Depois

Adicionado campo de data (input type=date); componente Reutilizável RecurrenceControls (toggle, tipo, quantidade, data de início); lógica de criação/atualização que calcula intervalo e data de fim, gera múltiplas entradas com valores divididos; edição pré‑preenche a data existente.

Benefícios

  • ✅ Usuário pode lançar entradas recorrentes configuráveis (mensal, trimestral, semestral, anual).
  • ✅ Valores e quantidades são distribuídos automaticamente entre as parcelas.
  • ✅ Edição de data preserva a intenção do lançamento (não reseta para hoje).
  • ✅ Código reaproveitado via função pure calculateRecurrence, facilitando manutenção.
  • ✅ Testes unitários cobrem cenários de recorrência e fallback para comportamento legado.
