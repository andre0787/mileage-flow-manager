🔴 Bugs Corrigidos

CRÍTICO Tela Preta — Root Cause
src/App.tsx
Problema: AppSidebar e BottomTabBar usavam useData() mas estavam FORA do DataProvider. Resultado: crash silencioso → tela preta após login.
Causa raiz: Hierarquia de providers incorreta no AppLayout. O DataProvider começou DENTRO do conteúdo, mas os componentes de navegação estavam do lado de fora.
Fix: DataProvider movido para envolver toda a árvore de componentes.
CRÍTICO Transfer Reversal — Custo Proporcional
src/hooks/useDatabase/entries.ts
Problema: Ao reverter/editar uma transferência, o código restaurava entry.amountPaid na conta origem. Mas em transferências, a origem teve debitado custo proporcional, não o total pago. Isso distorcia o investimento da conta.
Fix: Agora calcula calcProportionalCost() antes de reverter, tanto em useUpdateEntryMutation quanto em useDeleteEntryMutation.
MÉDIO .sort() Mutava Arrays Memoizados
src/pages/Relatorios.tsx
Problema: ownerReports.sort() mutava o array original do useMemo. Pode causar re-renders perdidos ou comportamento imprevisível.
Fix: Alterado para [...ownerReports].sort() — cria cópia antes de ordenar.
MÉDIO clearAccountData Não Preservava Transferência
src/hooks/useDatabase/shared.ts
Problema: UI dizia "O tipo Transferência será preservado", mas o código deletava TUDO incluindo origem_types.
Fix: Após deletar todos os origem_types, o código agora re-insere o tipo built-in "Transferência".
MÉDIO window.location.href em vez de navigate
src/pages/Dashboard.tsx
Problema: window.location.href = '/entradas' causava reload completo da página em vez de navegação client-side.
Fix: Substituído por useNavigate() do React Router.
MÉDIO Import Morto de SkeletonHero
src/pages/Dashboard.tsx
Problema: SkeletonHero foi deletado na PR #65 mas o import continuava no Dashboard. Causava runtime error → tela preta.
Fix: Import removido.
BAIXO Supabase Client Crashava sem Env Vars
src/lib/supabase.ts
Problema: throw new Error() se VITE_SUPABASE_URL ou VITE_SUPABASE_ANON_KEY faltassem. Causava crash no import.
Fix: Agora cria client com placeholder e loga erro em vez de.throw.
BAIXO main.tsx Sem Error Handling
src/main.tsx
Problema: Se o render do root falhasse, nada era mostrado — só tela preta.
Fix: Adicionado try/catch com fallback visual que mostra erro e botão para limpar cache.

📚 Novas Convenções Adicionadas

1. Hierarquia de Providers — OBRIGATÓRIO
Toda componente que usa um Context DEVE estar dentro do Provider correspondente.
Previne: tela preta por crash de contexto.
2. Invariantes Financeiras — OBRIGATÓRIO
Toda operação que altera saldo DEVE ter uma inversão espelhada testada.
Previne: distorção de investimento em transfers/reversals.
3. Imutabilidade de Estado — OBRIGATÓRIO
Nunca mutar arrays ou objetos que vêm de useMemo ou useState.
Previne: comportamento imprevisível em re-renders.
4. Promessas de UI — OBRIGATÓRIO
Se a UI mostra uma mensagem ao usuário, o código DEVE cumprir a promessa.
Previne: UI enganosa (diz X, faz Y).
5. Config Global — NÃO DUPLICAR
Configurações definidas no QueryClient global não devem ser repetidas em queries individuais.
Previne: config duplicada e inconsistente.

💡 Lições Aprendidas

1. Ao deletar um export, sempre grep em todos os imports
grep -rn "NomeDoExport" src/ --include="*.tsx" --include="*.ts"
2. Hierarquia de providers é invisível mas crítica

Componentes que usam Context precisam estar na sub-árvore do Provider. Uma linha fora = crash silencioso.

3. Deploy automático pode falhar silenciosamente

Vercel às vezes não aciona build no push. Ter vercel --prod como fallback é essencial.

4. Error handler global salva vidas

window.onerror com fallback visual Transforma tela preta em mensagem acionável.

5. Service Worker é敦 para debug

SW cacheando versão antiga = impossível ver mudanças. Version check no localStorage é a solução mais simples.
