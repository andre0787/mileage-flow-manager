🔐 Autenticação

Público Rotas sem autenticação

1. Cadastro de Usuário

/login → Modal de cadastro

Fluxo Principal
  1. Usuário clica em "Cadastre-se"
  2. Preenche: #name, #email, #password
  3. Submete formulário
  4. Recebe confirmação
  5. Login automático → redireciona para /
Edge Cases: Email já cadastrado • Senha < 6 caracteres • Campos obrigatórios vazios • Rede cai durante cadastro
Teste: Cadastrar usuário novo → verificar redirecionamento para dashboard
2. Login

/login

Fluxo Principal
  1. Usuário insere email e senha
  2. Clica em "Entrar"
  3. Redireciona para /
Edge Cases: Credenciais inválidas • Usuário não existe • Token expirado • Conta bloqueada
Teste: Login com credenciais válidas → verificar acesso ao dashboard
Teste Crítico: Login com senha errada → verificar mensagem de erro amigável
3. Recuperação de Senha

/forgot-password → /reset-password

Fluxo Principal
  1. Usuário insere email em /forgot-password
  2. Clica em "Enviar link"
  3. Recebe email com link
  4. Clica no link → redireciona para /reset-password
  5. Insere nova senha
  6. Confirma → senha atualizada
Edge Cases: Email não cadastrado (mensagem genérica) • Token expirado (24h) • Senhas não coincidem • Token já utilizado
Teste: Solicitar recuperação → verificar email recebido

📥 Entradas

Protegido /entradas CRUD

5. Lista de Entradas
Componentes
  • Abas Milhas/Pontos
  • Tabela de entradas com paginação
  • Botão "Nova Entrada"
  • Filtros (período, programa)
Teste: Lista com 25+ entradas → verificar paginação (20 itens/página)
6. Criar Entrada
Fluxo Principal
  1. Clica em "Nova Entrada"
  2. Drawer abre à direita
  3. Preenche: #accountId, #origemTypeId, #amount, #amountPaid, #date
  4. Clica em "Salvar"
  5. Toast.success → lista atualizada
Edge Cases: Campos obrigatórios • Valores numéricos inválidos • Conta não existe • Rede cai
Teste Crítico: Criar entrada → verificar saldo da conta atualizado
7. Editar Entrada
Fluxo Principal
  1. Clica em "Editar" na tabela
  2. Drawer abre com dados preenchidos
  3. Altera campos
  4. Clica em "Salvar Alterações"
  5. Toast.success → lista atualizada
Teste Crítico: Editar valor → verificar recálculo de custo médio
8. Excluir Entrada
Fluxo Principal
  1. Clica em "Excluir" na tabela
  2. AlertDialog aparece confirmando
  3. Usuário confirma
  4. Toast.success → lista atualizada
Teste Crítico: Excluir entrada → verificar reversão do saldo da conta
9. Transferência

Fluxo Especial

Fluxo Principal
  1. Seleciona tipo "Transferência"
  2. Preenche: conta origem, conta destino, quantidade
  3. Sistema calcula custo automaticamente
  4. Cria entrada de saída na origem
  5. Cria entrada de entrada no destino
Edge Cases: Saldo insuficiente • Mesma conta origem/destino • Transferência parcial
Teste Crítico: Transferência → verificar saldo origem diminuiu e destino aumentou
